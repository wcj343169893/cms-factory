Easy Widgets jQuery plugin - Example 16

This example show how to use "onChangePositions" and "onRefreshPositions" callbacks. This callbacks permit you to save the Widgets positions in a server database, and return it after to repositioned the Widgets accordly.

The plugin can be use the Cookies feature to save the positions of the Widgets and retrieve after to repositioned, but, if you preffer, you can save the appropiate serialized string in a database, and use this to repositioned the Widgets.

The plugin call the "onChangePositions" when a Widget change the position, and send to you an argument, that contain the positions of all the Widgets, in a string with the appropiate formate, and ready to save in the DB.

Is the same string that the plugin save in a cookie, if is use. Well. You save the string in a database, and when the page are reload, the plugin execute the "onRefreshPositions", that expect as result the string that you previously saved in the database, that is, the result of the other callback "onChangePositions".

When the plugin receive the appropiate string from the "onRefreshPositions" callback, the Widgets are repositioned. This occurr every time that a page is loaded. Of course, you can retrive the serialized string from a database or for other place: the point is that "onRefreshPositions" expect the string that you obtain from "onChangePositions".
